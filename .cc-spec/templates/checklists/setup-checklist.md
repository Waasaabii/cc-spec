# 环境配置检查清单

**用途**: 验证项目环境配置和依赖安装是否正确
**适用场景**: 新项目初始化、环境迁移、依赖更新后

## 环境检查

- [ ] ENV001 Python/Node.js 版本符合项目要求
- [ ] ENV002 必需的系统依赖已安装（如数据库客户端、编译工具等）
- [ ] ENV003 环境变量已正确配置
- [ ] ENV004 开发工具（IDE、编辑器）已安装并配置

## 依赖管理

- [ ] DEP001 项目依赖已完整安装（requirements.txt / package.json）
- [ ] DEP002 依赖版本与 lock 文件一致
- [ ] DEP003 可选依赖已根据需求安装
- [ ] DEP004 依赖包没有已知的安全漏洞

## 配置文件

- [ ] CFG001 配置文件已从模板复制（如 .env.example → .env）
- [ ] CFG002 数据库连接配置正确
- [ ] CFG003 第三方服务 API 密钥已配置
- [ ] CFG004 敏感配置文件已添加到 .gitignore

## 服务启动

- [ ] SVC001 本地开发服务器可以正常启动
- [ ] SVC002 数据库连接测试通过
- [ ] SVC003 测试套件可以正常运行
- [ ] SVC004 构建命令可以成功执行

## 注意事项

- 配置敏感信息时请勿泄露到版本控制
- 定期检查依赖安全更新
- 记录特殊的环境配置要求
