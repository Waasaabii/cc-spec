## 执行方式

执行文档支持 **主 Agent + SubAgent** 并行开发模式：

```
主 Agent 职责:
├── 读取执行文档
├── 生成变更摘要
├── 分发任务到 SubAgent（最多 15 并行）
├── 收集结果 + 技术检查
└── 更新状态

SubAgent 职责:
├── 接收精简上下文（~500 tokens）
├── 读取目标文件
├── 执行开发任务
└── 返回结果
```

## 文档规范

每个执行文档应包含：

1. **架构说明** - 主 Agent + SubAgent 执行模式
2. **上下文预算** - 确保主 Agent ≤150K tokens
3. **预估总览** - 每个任务的上下文预估
4. **任务依赖图** - 可视化依赖关系
5. **任务详情** - 每个任务的必读文件、输出文件、Checklist
6. **执行计划** - 分阶段执行方案
7. **质量验收** - 功能和质量标准
